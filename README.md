# GuideFlow Knowledge Base

GuideFlow를 만들며 실제로 사용한 기술과 AI 협업 노하우를 다음 프로그램에서도 바로 재사용할 수 있게 정리한 저장소입니다.

사이트: https://ai-development-wiki.vercel.app

## 처음에는 목적에 맞는 입구 하나만 고르세요

1. [GUIDEFLOW_CONTEXT.md](GUIDEFLOW_CONTEXT.md) — GuideFlow의 목적과 작업 방식
2. [TECH_INDEX.md](TECH_INDEX.md) — 프로그램을 만들기 전에 준비할 도구와 사용 이유
3. [AI_RULES.md](AI_RULES.md) — AI에게 일을 맡기고 결과를 확인하는 방법
4. [CONCEPT_GUIDE.md](CONCEPT_GUIDE.md) — CLI, API, 플러그인, MCP, 스킬, 에이전트 같은 용어 설명
5. [AUTOMATION_GUIDE.md](AUTOMATION_GUIDE.md) — 자동화 수준과 실제 작업 흐름
6. [AI_TOOL_MAP.md](AI_TOOL_MAP.md) — ChatGPT, Claude, Gemini, Codex와 제작 도구 선택 가이드

## 이 저장소가 남기는 두 가지

### GuideFlow 개발 기술

기술 이름을 외우기 위한 문서가 아닙니다. 이미지 붙여넣기, 교체, 드래그, 장면 관리, 재생, 내보내기와 배포가 왜 필요했고 어디에서 문제가 생겼는지 초보자 기준으로 설명합니다.

### AI 협업 노하우

GuideFlow 개발 중 실제로 겪은 실패를 다음 프로젝트에서 반복하지 않기 위한 규칙입니다. 기술 방향 변경, 불필요한 설치, 범위 확대, 기존 기능 손상과 확인하지 않은 완료 보고를 막는 요청문을 제공합니다.

## 사이트의 다섯 가지 입구

- **준비하기** — 무엇을 설치하고 어떤 자료를 AI에게 줘야 하는지 확인합니다.
- **AI에게 맡기기** — 기능 요청, 오류 전달, 검증 요청 방법을 배웁니다.
- **도구 고르기** — 만들려는 프로그램에 맞는 AI와 제작 도구를 찾습니다.
- **용어 이해하기** — 낯선 개발·AI 용어를 쉬운 비유와 역할 구분으로 이해합니다.
- **자동화 설계하기** — 사람과 AI가 각각 맡을 범위를 단계별로 결정합니다.

## 완료 기준

- 처음 방문한 사람이 사이트의 목적을 바로 이해합니다.
- Start Here에서 프로젝트 전체 맥락을 읽고 복사하거나 저장할 수 있습니다.
- GuideFlow 기술과 AI 협업 노하우를 카드로 찾아볼 수 있습니다.
- PC와 모바일에서 핵심 동선이 깨지지 않습니다.
- 구현, GitHub 커밋, 배포와 브라우저 동작을 각각 확인합니다.

예전 위키 실험 자료는 저장소 이력에 남아 있지만, 현재 사이트의 주요 화면에는 노출하지 않습니다.
