# 자동화 설계 가이드

완전 자동화를 목표로 기능을 무겁게 만들지 않고, 반복되는 기본 틀을 AI가 처리한 뒤 사람이 최종 판단하도록 설계합니다.

## 자동화 다섯 단계

### 1단계 — AI가 초안 만들기

사람이 요청하고 AI가 한 번의 결과를 만듭니다.

- 사용자: 목표와 자료 제공, 결과 선택
- AI: 기획·문서·코드 초안
- 예시: GuideFlow 화면 시안 만들기
- 주의: 결과를 그대로 사용하지 말고 방향부터 확인

### 2단계 — 버튼 한 번으로 여러 단계 실행

정해진 작업 묶음을 사용자가 필요할 때 시작합니다.

- 사용자: 실행 버튼과 입력 자료 선택
- AI: 여러 단계를 순서대로 실행
- 예시: 이미지 여러 장 정리하고 미리보기 생성
- 주의: 각 단계의 실패 위치와 재실행 방법 필요

### 3단계 — API로 반복 작업 연결

프로그램이 AI·음성·이미지 서비스를 자동 호출합니다.

- 사용자: 비용 한도와 결과 기준 관리
- AI: 호출 코드·오류 처리·기록
- 예시: 대본마다 이미지와 TTS 초안 생성
- 주의: API 키 보호와 호출 비용 제한 필요

### 4단계 — 예약·조건 자동 실행

시간이나 사건을 기준으로 사람이 누르지 않아도 실행합니다.

- 사용자: 실행 조건과 중지 방법 결정
- AI: 스케줄·모니터링·알림 구성
- 예시: 매일 자료 수집 후 초안 보고서 생성
- 주의: 잘못된 반복 실행과 누적 비용 감시 필요

### 5단계 — 사람이 최종 검수

자동 결과를 사람이 선택·수정·승인한 뒤 사용합니다.

- 사용자: 품질·저작권·사실·사용감 최종 판단
- AI: 검수 자료와 수정 후보 제공
- 예시: 초벌 영상을 편집 프로그램에서 마무리
- 주의: 완전 자동화를 목표로 검수 단계를 없애지 않기

## 실전 작업 흐름

### 🎞️ GuideFlow 아이디어 스케치

아이디어 → 캡처 붙여넣기 → 순서·시간 조정 → 재생 확인 → 사람이 본편집

- 도구: ChatGPT + Codex + 브라우저
- 원칙: 5분 안에 초벌 가이드를 만들고 완성 편집은 기존 편집기에서 합니다.

### 🌐 웹 프로그램 만들기

사용 흐름 정리 → 작은 화면 구현 → Git 기록 → 공개 배포 → 브라우저 검증

- 도구: ChatGPT 또는 Claude + Codex 또는 Claude Code + GitHub + Vercel
- 원칙: 코드 생성보다 공개 주소에서 실제로 되는지 확인하는 단계가 중요합니다.

### 🎬 영상 초벌 자동화

자료 조사 → 대본 → 이미지·영상 → TTS → 자막·조립 → 사람이 최종 편집

- 도구: 자료 조사 AI + 이미지·영상 도구 + TTS + Whisper + FFmpeg
- 원칙: 100% 자동 완성보다 반복되는 기본 틀을 자동화하고 사람이 품질을 마무리합니다.

### 📚 자료 기반 콘텐츠

공식 자료 수집 → 근거 정리 → 구성안 → 초안 → 출처·사실 검수

- 도구: NotebookLM + Deep Research + ChatGPT 또는 Claude
- 원칙: 커뮤니티 의견과 사실을 구분하고 공식·1차 자료를 우선합니다.

### 🔁 반복 업무 자동화

수동으로 한 번 성공 → 순서 기록 → API·MCP 연결 → 예약 실행 → 실패 알림·중지

- 도구: Skills + MCP + n8n 또는 Automations + Windows 작업 스케줄러
- 원칙: 검증되지 않은 작업을 바로 예약하지 않고 먼저 수동으로 성공시킵니다.
