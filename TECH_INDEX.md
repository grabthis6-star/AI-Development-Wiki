# GuideFlow 기술 색인

GuideFlow를 만들 때 실제로 사용한 기술을 비개발자도 다시 활용할 수 있도록 정리한 문서입니다.

## 이 문서를 보는 방법

각 기술은 다음 질문에 답하도록 설명합니다.

1. 이 기술은 무엇인가?
2. 왜 필요했는가?
3. GuideFlow의 어디에 사용했는가?
4. 쉽게 비유하면 무엇인가?
5. 어떤 문제가 있었고 어떻게 해결했는가?
6. 다른 프로그램에서는 어떻게 재사용하는가?

## 화면을 만드는 기술

### HTML
버튼, 이미지, 장면 목록처럼 화면에 보이는 요소의 구조를 만듭니다. 건물로 비유하면 뼈대입니다.

### CSS
색, 크기, 간격, 배치와 모바일 화면을 정합니다. 같은 HTML도 CSS에 따라 전혀 다른 제품처럼 보입니다.

### JavaScript
버튼 클릭, 이미지 붙여넣기, 장면 추가·삭제·정렬과 같은 실제 동작을 처리합니다.

### DOM
JavaScript가 화면 요소를 찾고 변경할 수 있게 해주는 브라우저의 화면 지도입니다.

### Event Listener
클릭, 키보드 입력, 붙여넣기, 드래그처럼 사용자의 행동을 감지합니다.

## GuideFlow 핵심 기능에 사용한 기술

### Clipboard API와 paste 이벤트
캡처한 이미지를 Ctrl+V로 바로 가져오는 핵심 기능입니다. 브라우저가 전달한 클립보드 항목에서 이미지 파일을 찾아 화면에 추가합니다.

### FileReader와 Object URL
사용자가 붙여넣거나 선택한 이미지 파일을 브라우저 화면에서 볼 수 있는 주소로 바꿉니다. 이미지 크기와 보관 기간에 따라 적절한 방식을 골라야 합니다.

### Pointer Event
마우스, 터치, 펜 입력을 하나의 방식으로 처리합니다. PC와 모바일에서 같은 드래그 경험을 만들 때 사용합니다.

### 상태 관리
현재 선택된 장면, 이미지 목록, 장면 순서와 재생 상태를 한곳에서 관리하는 방식입니다. 데이터가 바뀌면 화면도 같은 내용을 보여줘야 합니다.

### 장면 번호와 고유 ID
화면에 보이는 순서 번호와 내부 식별자를 분리합니다. 장면을 이동하거나 삭제해도 다른 장면을 잘못 수정하지 않게 합니다.

### 이미지 교체
위치와 크기는 유지하고 이미지 내용만 바꿉니다. 액자는 그대로 두고 사진만 바꾸는 것과 같습니다.

### 재생 타이머
각 장면의 표시 시간을 기준으로 다음 장면을 보여줍니다. 재생 시작, 일시정지, 종료와 현재 장면을 구분해 관리해야 합니다.

### LocalStorage
브라우저에 작은 설정이나 임시 상태를 저장합니다. 큰 이미지나 중요한 영구 프로젝트 저장에는 적합하지 않습니다.

## 개발과 배포에 사용한 도구

### Cursor 또는 Codex
코드를 직접 모두 작성하기보다 저장소를 읽고 작은 변경을 구현하도록 AI와 협업하는 도구입니다.

### Node.js와 npm
개발 도구를 실행하고 필요한 패키지를 설치하며 빌드 명령을 실행합니다. 완성된 정적 웹사이트를 보는 사용자에게는 보통 필요하지 않습니다.

### Git
변경 이력을 컴퓨터에서 관리합니다. 문제가 생겼을 때 어떤 변경에서 시작됐는지 확인할 수 있습니다.

### GitHub
원본 코드와 커밋을 보관하고 협업과 배포의 기준점으로 사용합니다.

### Vercel
GitHub의 웹 프로젝트를 인터넷 주소로 배포합니다. GitHub 커밋 성공, Vercel 배포 성공, 실제 브라우저 동작은 서로 다른 검증 단계입니다.

## 다음 프로젝트에서 먼저 고를 것

- 웹 화면이 필요한가: HTML, CSS
- 사용자의 조작이 필요한가: JavaScript, Event
- 붙여넣기가 핵심인가: Clipboard API
- 드래그가 필요한가: Pointer Event
- 데이터를 잠깐 저장하는가: LocalStorage
- 영구 저장과 공유가 필요한가: 서버 또는 데이터베이스
- 인터넷에 공개할 것인가: GitHub와 배포 서비스
