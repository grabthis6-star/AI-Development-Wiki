# GuideFlow 기술 색인

개발 경험이 없는 사람도 AI와 협업해 프로그램을 만들 수 있도록, 사용자가 준비하고 전달하고 확인해야 할 일을 안내하는 실전 가이드

이 문서는 웹사이트의 단일 지식 데이터에서 자동 생성됩니다. 기술 이름만 나열하지 않고 GuideFlow에서 왜 사용했는지와 재사용 방법까지 함께 설명합니다.

## AI 작업 환경 선택하기

모든 도구를 설치할 필요는 없습니다. 대화·기획용 AI 하나와 실제 파일을 수정할 개발 도구 하나부터 선택하세요. 전체 비교는 AI_TOOL_MAP.md에서 확인할 수 있습니다.

### 📂 ChatGPT Projects
- 종류: OpenAI · AI 내부 기능
- 언제 사용: 같은 프로그램을 여러 채팅에 걸쳐 오래 만들거나 참고 문서를 계속 사용해야 할 때
- 사용자가 준비할 것: 프로젝트 목적, 참고 파일, 반복해서 지킬 작업 원칙
- 공식 안내: https://chatgpt.com/

### 🛠️ Codex
- 종류: OpenAI · 개발 에이전트
- 언제 사용: 실제 프로그램 기능 추가, 버그 수정, GitHub 반영과 브라우저 검증이 필요할 때
- 사용자가 준비할 것: 프로젝트 폴더, 컨텍스트 문서, 이번 작업, 유지할 기능, 완료 기준
- 공식 안내: https://developers.openai.com/codex/

### 📜 AGENTS.md
- 종류: OpenAI · Codex 기능
- 언제 사용: 같은 설명이나 실수가 반복되고, 실행·검증 방법을 모든 작업에 적용해야 할 때
- 사용자가 준비할 것: 폴더 구조, 실행 방법, 유지 원칙, 완료 기준
- 공식 안내: https://learn.chatgpt.com/docs/agent-configuration/agents-md

- 사용자가 준비할 것: 프로젝트 지식 파일과 공통 지침
- 공식 안내: https://support.anthropic.com/en/articles/9517075-what-are-projects

### 💎 Gemini Gems
- 종류: Google · AI 내부 기능
- 언제 사용: 같은 역할과 결과 형식으로 반복 질문하고 싶을 때
- 사용자가 준비할 것: 역할, 반복 작업, 참고 자료, 원하는 형식
- 공식 안내: https://support.google.com/gemini/answer/15235603

### 🧪 Google AI Studio

- 사용자가 준비할 것: Google 계정, 앱 아이디어, 사용할 AI 능력과 비용 범위
- 공식 안내: https://ai.google.dev/gemini-api/docs/aistudio-build-mode

## 1. 시작 전에 준비하기

AI에게 일을 맡기기 전에 작업 장소와 기본 도구를 준비합니다.

### 📁 프로젝트 폴더
- 한 줄 설명: AI가 파일을 읽고 수정할 수 있는 하나의 작업 공간
- GuideFlow 사용 위치: 프로그램을 처음 시작할 때 가장 먼저 만듭니다. GuideFlow라면 GuideFlow라는 폴더 하나가 프로젝트 전체입니다.
- 사용 이유: 이미지·코드·설명 문서가 흩어지면 AI도 현재 상태를 정확히 파악하기 어렵습니다.
- 다른 프로젝트 재사용: 앞으로 만드는 모든 프로그램은 프로젝트별 폴더 하나에서 시작합니다.
- 확인 항목: 프로젝트 이름의 폴더가 하나 있다. / AI가 그 폴더 안의 파일 목록을 볼 수 있다. / 다운로드한 자료도 프로젝트 안에 정리했다.

### 🤖 ChatGPT와 Codex
- 한 줄 설명: 아이디어 대화와 실제 파일 작업을 나누어 맡기는 AI 도구
- GuideFlow 사용 위치: ChatGPT에서는 아이디어와 사용 흐름을 정리하고, Codex에서는 실제 프로젝트 폴더를 열어 코드 수정과 검증을 맡깁니다.
- 사용 이유: 채팅으로 코드 조각만 받으면 사용자가 직접 파일을 찾고 붙여넣어야 합니다. Codex는 프로젝트 파일을 직접 확인하고 수정할 수 있습니다.
- 다른 프로젝트 재사용: 새 프로젝트마다 PROJECT_CONTEXT.md 하나를 만들어 같은 방식으로 시작합니다.
- 확인 항목: Codex에서 올바른 프로젝트 폴더를 열었다. / AI가 컨텍스트 문서를 먼저 읽었다. / 이번 작업을 기능 하나로 제한했다.

### 📝 Cursor 또는 코드 편집기
- 한 줄 설명: 프로젝트 파일을 눈으로 확인하고 AI 수정 결과를 살펴보는 프로그램
- GuideFlow 사용 위치: Codex가 직접 작업하는 경우 필수는 아니지만 파일을 열어보거나 간단히 실행할 때 유용합니다.
- 사용 이유: 사용자가 코드를 작성하기 위해서가 아니라 어떤 파일이 있는지, AI가 무엇을 바꿨는지 확인하기 위해 사용합니다.
- 다른 프로젝트 재사용: 웹사이트, 데스크톱 앱, 자동화 도구 등 대부분의 프로젝트에서 사용합니다.
- 확인 항목: Cursor 또는 VS Code 중 하나만 설치했다. / 프로젝트 폴더를 열 수 있다. / 코드를 이해하려고 멈추지 않고 AI에게 설명을 요청한다.

## 2. 기본 개발 도구 이해하기

왜 설치하는지 알고, 설치 후 정상인지 확인하는 데 필요한 만큼만 배웁니다.

### 🟢 Node.js
- 한 줄 설명: 웹 프로그램을 개발하고 실행하는 데 사용하는 작업 엔진
- GuideFlow 사용 위치: React·Vite 같은 웹 프로젝트를 만들거나 개발 화면을 실행할 때 사용합니다. 완성된 웹사이트 사용자는 보통 설치할 필요가 없습니다.
- 사용 이유: 프로젝트에 필요한 부품을 내려받고(npm), 개발 화면을 실행하고, 배포용 파일을 만드는 역할을 합니다.
- 다른 프로젝트 재사용: React, Vite, Electron 등 JavaScript 기반 프로그램 제작에 사용합니다.
- 확인 항목: node -v에 버전이 나온다. / npm -v 또는 npm.cmd -v에 버전이 나온다. / 오류가 나면 마지막 줄만이 아니라 전체를 AI에게 전달한다.

### 🌿 Git과 GitHub
- 한 줄 설명: 변경 기록을 남기는 도구와 그 기록을 보관하는 온라인 저장소
- GuideFlow 사용 위치: AI가 수정한 내용을 이전 상태와 비교하고, 문제가 생기면 어떤 변경에서 시작됐는지 찾을 때 사용합니다.
- 사용 이유: AI가 여러 파일을 수정해도 버전이 남아 안전하게 실험할 수 있고 Vercel 같은 배포 서비스와 연결할 수 있습니다.
- 다른 프로젝트 재사용: 모든 프로그램의 백업·협업·배포 과정에 재사용합니다.
- 확인 항목: git --version이 표시된다. / GitHub 저장소 주소를 알고 있다. / AI가 최신 커밋 주소를 알려준다.

### 🚀 Vercel과 공개 주소
- 한 줄 설명: 만든 웹 프로그램을 인터넷에서 확인할 수 있게 배포하는 서비스
- GuideFlow 사용 위치: GuideFlow나 위키처럼 브라우저에서 쓰는 프로그램을 다른 컴퓨터에서도 열어볼 때 사용합니다.
- 사용 이유: 내 컴퓨터에서만 되는지, 실제 사용자 환경에서도 되는지 확인할 수 있습니다.
- 다른 프로젝트 재사용: 정적 사이트와 다양한 웹 앱을 빠르게 공유할 때 사용합니다.
- 확인 항목: Vercel 상태가 Success다. / 배포가 최신 커밋과 연결됐다. / 공개 주소에서 새 기능을 직접 확인했다.

## 3. 필요할 때만 추가하기

Python·FFmpeg 같은 도구는 기능이 실제로 필요할 때만 설치합니다.

### 🐍 Python — 필요할 때만
- 한 줄 설명: 자동화·AI 처리·문서 제작 등에 자주 쓰이는 별도의 프로그래밍 실행 환경
- GuideFlow 사용 위치: 이미지 일괄 처리, 데이터 분석, PDF·문서 자동 생성이나 Python 서버가 필요할 때만 설치합니다.
- 사용 이유: 웹 화면만 만드는 프로젝트라면 Node.js만으로 충분할 수 있습니다. AI가 Python을 좋아한다는 이유만으로 설치하지 않습니다.
- 다른 프로젝트 재사용: 파일 자동화, 데이터 처리, AI 모델과 문서 생성 프로젝트에서 사용합니다.
- 확인 항목: Python이 꼭 필요한 기능을 설명할 수 있다. / 공식 사이트에서 설치했다. / python --version이 나온다.

### 🎬 FFmpeg — 영상 출력이 필요할 때
- 한 줄 설명: 이미지·영상·소리를 변환하고 MP4 같은 결과물을 만드는 전문 도구
- GuideFlow 사용 위치: GuideFlow에서 실제 MP4 또는 GIF 내보내기가 필요해질 때 사용합니다. 이미지 배치만 시험하는 초기 버전에는 필요 없습니다.
- 사용 이유: 브라우저 화면만으로 어려운 영상 인코딩을 안정적으로 처리합니다.
- 다른 프로젝트 재사용: 썸네일 추출, 영상 변환, 오디오 결합과 짧은 미리보기 제작에 사용합니다.
- 확인 항목: 내보내기가 지금 꼭 필요한지 확인했다. / ffmpeg -version이 나온다. / 짧은 샘플 파일부터 시험했다.

## 4. AI에게 정확히 전달하기

코드 대신 아이디어, 현재 화면, 오류와 완료 기준을 전달합니다.

### 📸 스크린샷·오류·기대 결과
- 한 줄 설명: 사용자가 AI에게 제공해야 하는 가장 중요한 세 가지 자료
- GuideFlow 사용 위치: 화면이 다르거나 오류가 생겼을 때, 그리고 디자인을 수정하고 싶을 때 제공합니다.
- 사용 이유: AI는 사용자가 보는 화면과 마음속 기대 결과를 자동으로 알 수 없습니다. 자료가 구체적일수록 추측이 줄어듭니다.
- 다른 프로젝트 재사용: 버그 수정, 디자인 피드백, 설치 오류와 배포 문제에 모두 사용합니다.
- 확인 항목: 문제가 보이는 전체 화면을 첨부했다. / 문제가 생기는 순서를 적었다. / 실제 결과와 원하는 결과를 구분했다.
