# Prompt Library

아래 문장은 새 채팅이나 다른 프로젝트에서도 바로 복사해 사용할 수 있다.

## 새 프로젝트 시작

```text
아래 AI Development Wiki의 시작 문서와 작업 규칙을 먼저 읽고 기준으로 삼아줘.
https://ai-development-wiki.vercel.app/#/start-here

프로그램 목적:
[누가 어떤 상황에서 무엇을 쉽게 하게 만들지 작성]

사용자는 비개발자다. 쉬운 말로 설명하고, 가장 작은 핵심 기능부터 구현해. 확인하지 않은 GitHub 반영이나 배포 결과를 완료라고 말하지 마.
```

## 스프린트 진행

```text
서로 연관된 여러 변경을 하나의 스프린트로 묶어 진행해. 중간의 사소한 수정은 보고하지 말고, 실제 구현과 GitHub 반영을 끝낸 뒤 아래만 보고해.

1. 추가된 기능
2. 변경된 파일과 커밋
3. 실제로 검증한 결과
4. 아직 검증하지 못한 부분
```

## 버그 수정

```text
현재 보이는 현상과 기대 결과를 먼저 비교해. 관련 파일과 직전 변경을 확인하고, 추측으로 원인을 단정하지 마. 가장 작은 수정으로 고친 뒤 재현 절차와 검증 결과를 알려줘.
```

## UI 개선

```text
기존 기능을 깨뜨리지 말고 사용자가 가장 자주 수행하는 흐름을 더 단순하게 만들어. 데스크톱과 모바일, 키보드 조작, 빈 상태와 오류 상태를 함께 확인해.
```

## 코드 리뷰

```text
다음 순서로 검토해.
1. 실제 오류 가능성
2. 데이터 손실 또는 보안 위험
3. 유지보수성
4. 성능
5. 사용성

반드시 수정할 항목과 선택 개선을 나누고, 근거가 없는 문제는 만들지 마.
```

## 배포 문제

```text
코드, 빌드 설정, 출력 경로, 환경 변수, 배포 상태, 실제 도메인을 분리해서 확인해. GitHub 변경이 있다는 이유만으로 배포 완료라고 판단하지 마.
```

## 작업 기록

```text
이번 작업에서 다음 프로젝트에도 재사용할 내용을 위키 문서로 정리해.
- 목표
- 변경 내용
- 실패한 시도
- 실제 원인
- 해결 방법
- 검증 방법
- 다음 체크리스트
```
