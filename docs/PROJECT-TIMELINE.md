# Project Timeline

프로젝트의 기능 목록이 아니라 중요한 결정과 배움의 흐름을 시간 순서로 기록한다.

## GuideFlow → AI Development OS

### 1. GuideFlow 시작

- 스토리보드 제작 흐름을 더 단순하게 만드는 것이 목적이었다.
- 실제 사용자가 자주 반복하는 장면 편집 작업을 우선했다.

### 2. 핵심 편집 기능 확장

- Scene Numbering
- Duplicate Scene
- Image Replace
- Pointer Event
- 장면 단위 작업 흐름 개선

### 3. 저장과 배포 체계

- GitHub를 코드 원본과 변경 기록의 기준으로 사용했다.
- Vercel을 실제 사용자 화면의 검증 장소로 사용했다.
- 코드 변경과 실제 배포 완료를 서로 다른 단계로 관리했다.

### 4. 문제 해결 방식 정착

- 보이는 현상과 기대 결과를 먼저 비교했다.
- 직전 변경과 관련 파일을 확인했다.
- 수정 후 GitHub, 배포, 실제 화면을 차례로 검증했다.
- 확인하지 않은 결과를 완료라고 말하지 않는 원칙을 세웠다.

### 5. AI Development Wiki 제작

- 프로젝트마다 같은 설명을 반복하는 문제를 줄이기 위해 시작했다.
- 개발 도구, 협업 규칙, 문제 해결, 사례를 재사용 가능한 문서로 저장했다.

### 6. AI Development OS 단계

- 문서만 저장하는 위키에서 프롬프트, 패턴, 프로젝트 기억, 템플릿을 연결하는 시스템으로 확장한다.
- 새 AI가 짧은 시간 안에 사용자의 작업 방식을 이해하도록 만드는 것이 목표다.
