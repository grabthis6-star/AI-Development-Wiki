# AI Project Template

새 프로그램을 시작할 때 아래 파일과 기준을 복사해 사용한다.

## 필수 문서

```text
README.md
START-HERE.md
PROJECT-RULES.md
DECISIONS.md
TROUBLESHOOTING.md
```

## README.md

```markdown
# 프로젝트 이름

## 한 줄 목적
누가 어떤 상황에서 무엇을 더 쉽게 하게 만드는지 작성한다.

## 실제 사이트
배포 후 주소를 작성한다.

## 핵심 기능
- 기능 1
- 기능 2
- 기능 3

## 실행 방법
개발 서버와 빌드 명령을 작성한다.

## 저장과 배포
GitHub 저장소, 기본 브랜치, 배포 서비스를 작성한다.
```

## START-HERE.md

```markdown
# Start Here

- 사용자는 비개발자다.
- 쉬운 말로 설명한다.
- 목적과 완료 기준을 먼저 확인한다.
- 서로 연관된 작업은 스프린트로 묶는다.
- 확인하지 않은 구현, 커밋, 배포를 완료라고 말하지 않는다.
- 중요한 결정과 문제 해결은 문서에 남긴다.
```

## PROJECT-RULES.md

반드시 지킬 기술 및 제품 규칙을 기록한다.

- 데이터 저장 위치
- 사용 중인 프레임워크
- 배포 방식
- 디자인 원칙
- 모바일 지원 범위
- 금지된 변경
- 완료 기준

## DECISIONS.md

```markdown
## 날짜 — 결정 제목

- 상황:
- 선택한 방법:
- 다른 선택지:
- 선택 이유:
- 실제 결과:
- 다시 검토할 조건:
```

## 첫 스프린트 체크리스트

1. 목적을 한 문장으로 확정한다.
2. 첫 사용자가 완료해야 할 핵심 흐름을 고른다.
3. 최소 기능 1~3개만 구현한다.
4. GitHub에 원본을 저장한다.
5. Vercel 또는 선택한 서비스에 배포한다.
6. 실제 주소에서 핵심 흐름을 테스트한다.
7. 실패 원인과 배운 점을 기록한다.
