# 처음 시작하기

## 한 줄 설명

새로운 프로그램을 시작할 때 사용자와 AI가 같은 기준으로 일하도록 알려주는 출발 문서입니다.

## 왜 필요한가

새 채팅은 이전 채팅의 모든 내용을 자동으로 알지 못합니다. 이 문서를 먼저 읽히면 사용자의 수준, 작업 방식, 저장과 배포 규칙을 긴 설명 없이 빠르게 전달할 수 있습니다.

## 새 프로젝트 시작 순서

1. 만들 프로그램이 해결할 문제를 한 문장으로 적습니다.
2. 가장 먼저 필요한 핵심 기능을 한 가지 정합니다.
3. 이 문서와 프로그램 목적을 새 채팅의 AI에게 전달합니다.
4. GitHub 저장소를 만들고 원본을 보관합니다.
5. 웹서비스라면 Vercel 같은 배포 서비스를 연결합니다.
6. 실제 주소에서 동작을 확인합니다.
7. 새로 배운 내용과 문제 해결법을 위키에 기록합니다.

## 새 채팅에 붙여 넣는 시작 문구

```text
아래 AI Development Wiki의 시작 문서를 먼저 읽고 그 규칙을 기준으로 작업해.
https://ai-development-wiki.vercel.app/#/start-here

이번 프로그램의 목적:
[한두 문장으로 작성]

가장 먼저 필요한 기능:
[한 문장으로 작성]

사용자는 비개발자이므로 쉬운 말로 설명해. 확인하지 않은 변경은 완료됐다고 말하지 말고, 실제 결과를 기준으로 진행해.
```

## AI가 따라야 할 기본 규칙

- 어려운 개발 용어를 먼저 사용하지 않습니다.
- 필요한 경우 쉬운 비유를 먼저 말하고 정확한 용어를 덧붙입니다.
- 기존 기능을 확인하지 않고 무작정 바꾸지 않습니다.
- GitHub 변경은 실제로 확인한 뒤 반영됐다고 말합니다.
- 배포 성공과 실제 사이트 동작을 구분합니다.
- 큰 작업은 위험도에 따라 나눕니다.
- 단순 문서 추가와 안전한 디자인 개선은 여러 개 묶어 진행할 수 있습니다.
- 중요한 결정과 오류 해결법은 위키에 남깁니다.

## 사용자와 AI의 역할

### 사용자

- 무엇을 만들지 결정합니다.
- 실제 사용 느낌을 판단합니다.
- 우선순위와 완료 기준을 정합니다.

### AI

- 설계와 구현을 담당합니다.
- 관련 파일과 현재 상태를 확인합니다.
- 오류 원인을 조사하고 수정합니다.
- 작업 결과와 확인 방법을 보고합니다.
- 재사용할 지식을 위키 문서로 남깁니다.

## 이것만 기억하자

> 새 프로젝트 시작 = 시작 문서 주소 + 프로그램 목적 한두 문장 + 첫 번째 핵심 기능
