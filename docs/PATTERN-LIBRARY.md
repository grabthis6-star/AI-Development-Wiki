# Pattern Library

패턴은 코드 조각만 저장하지 않는다. 어떤 문제를 해결하는지, 언제 쓰는지, 완료 기준과 실패 조건까지 함께 기록한다.

## 검색

### 해결할 문제
사용자가 문서 제목을 정확히 기억하지 못해도 원하는 내용을 찾게 한다.

### 기본 구조
- 검색어를 소문자로 정규화한다.
- 제목, 설명, 태그, 본문을 함께 검색한다.
- 카테고리 필터와 검색어를 동시에 적용한다.
- 결과가 없을 때 검색어를 줄이도록 안내한다.

### 완료 기준
- 부분 단어로도 관련 문서가 나온다.
- 검색어를 지우면 전체 목록으로 돌아온다.
- 모바일에서도 입력창이 화면 밖으로 나가지 않는다.

## 사이드바 탐색

### 구성
- 모든 문서
- 카테고리
- 즐겨찾기
- 최근 본 문서
- 프로젝트 타임라인

### 반응형 규칙
데스크톱에서는 고정 탐색으로 사용하고, 좁은 화면에서는 본문 위의 일반 영역으로 바꾼다.

## 즐겨찾기와 최근 문서

- 개인 브라우저의 localStorage에 저장한다.
- 로그인이 없는 개인 위키에서 가볍게 사용할 수 있다.
- 저장 실패가 핵심 문서 열람을 막아서는 안 된다.

## 문서 목차

- 각 문서의 섹션 제목으로 자동 생성한다.
- 클릭하면 해당 섹션으로 이동한다.
- 모바일에서는 본문 위에 배치해 공간을 절약한다.

## 복사 가능한 프롬프트

- 프롬프트 원문과 설명을 분리한다.
- 복사 버튼을 제공한다.
- 복사 성공 상태를 짧게 표시한다.
- 클립보드 권한이 없을 때 오류를 숨기지 않는다.

## 자동 저장

- 입력할 때마다 즉시 서버에 요청하지 않는다.
- 짧은 지연 후 저장한다.
- 저장 중, 저장됨, 실패 상태를 구분한다.
- 실패해도 사용자가 작성한 내용을 즉시 지우지 않는다.

## 파일 업로드

- 업로드 전에 형식과 크기를 검사한다.
- 미리보기와 진행 상태를 제공한다.
- 실패 원인을 사용자 언어로 설명한다.
- 교체 작업은 기존 파일이 안전하게 유지되는지 확인한다.

## 모달

- ESC로 닫기
- 배경 클릭으로 닫기
- 키보드 포커스를 모달 안에 유지
- 닫은 뒤 원래 버튼으로 포커스 복귀
- 모달이 열린 동안 뒤 화면 스크롤 방지

## 문제 해결 기록

모든 패턴 실패는 다음 형식으로 남긴다.

```text
기대 결과
실제 현상
재현 순서
직전 변경
시도한 것
실제 원인
수정
검증
다음 체크리스트
```
