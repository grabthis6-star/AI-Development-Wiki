# AI Memory System

## 목적

AI Memory는 단순한 용어 설명이 아니라, 우리가 언제 어떤 프로젝트에서 무엇을 배웠고 왜 그런 결정을 했는지 다시 찾게 하는 기록 구조다.

## 기억 단위

각 기억은 아래 항목을 가진다.

- 날짜
- 프로젝트
- 주제
- 당시 상황
- 내린 결정
- 실제 결과
- 실패한 시도
- 다음에 재사용할 규칙
- 관련 문서와 커밋

## 기록 예시

```yaml
date: 2025
project: GuideFlow
topic: Vercel 배포
situation: 배포 주소에서 404가 발생함
decision: 코드만 보지 않고 빌드 설정과 경로를 분리해서 확인
result: 실제 배포 경로 문제를 찾아 수정
reusable_rule: 404는 코드, 빌드, 경로, 배포 상태를 각각 확인한다
```

## 검색 원칙

`React`를 검색하면 정의만 보여주는 것이 아니라 다음도 연결한다.

1. React를 처음 사용한 프로젝트
2. 사용한 이유
3. 당시 발생한 문제
4. 해결 방법
5. 관련 패턴과 프롬프트

## 운영 규칙

- 중요한 결정은 결과가 확인된 뒤 기록한다.
- 추측과 실제 원인을 분리한다.
- 같은 문제를 다시 겪었을 때 바로 행동할 수 있는 문장으로 끝낸다.
- 민감한 키, 비밀번호, 개인 정보는 저장하지 않는다.
