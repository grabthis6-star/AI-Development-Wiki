# Repository guidance

## Project purpose

This repository publishes the GuideFlow Knowledge Base: a beginner-friendly guide to building programs with AI, grounded in lessons learned while making GuideFlow.

## Source of truth

- `data/*.js` contains the structured knowledge data.
- `knowledge-data.js` is a re-export hub for the site and document generator.
- `index.html`, `styles.css`, and `app.js` are the only web entry files.
- `TECH_INDEX.md`, `AI_RULES.md`, `CONCEPT_GUIDE.md`, `AUTOMATION_GUIDE.md`, `PRACTICAL_GUIDE.md`, and `AI_TOOL_MAP.md` are generated by `generate-docs.mjs`.
- `GUIDEFLOW_CONTEXT.md` and `README.md` are maintained manually.

Do not edit a generated Markdown file without updating its source data first.

## Working rules

1. Read `GUIDEFLOW_CONTEXT.md` before changing product direction.
2. Keep the beginner audience and GuideFlow origin visible.
3. Make one focused change at a time and preserve existing routes.
4. Do not hard-code counts that can be derived from data.
5. Keep volatile product claims linked to an official source.
6. Report implementation, GitHub commit, deployment, and browser verification separately.

## Required checks

Run before publishing:

```bash
npm test
```

This validates structured data, builds the site, and checks the deployment bundle.
