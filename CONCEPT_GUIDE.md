# AI 개발 도구 용어 가이드

초보자가 AI에게 필요한 능력을 정확히 요청할 수 있도록 앱, CLI, API, 플러그인, MCP, 스킬과 에이전트의 차이를 설명합니다.

## 핵심 용어

### 🪟 GUI와 앱
- 한 줄 설명: 버튼과 메뉴가 보이는 일반적인 사용 화면
- 쉽게 비유하면: 카카오톡처럼 화면을 보며 클릭하는 방식입니다.
- 왜 필요한가: 초보자가 가장 쉽게 시작할 수 있지만, 큰 작업에서는 메모리 사용과 반복 클릭이 늘 수 있습니다.
- 사용자가 할 일: 앱을 열고 프로젝트를 선택하고 결과를 눈으로 확인합니다.
- AI가 할 일: 화면 안에서 대화하고 허용된 파일과 도구를 사용합니다.
- 추천 상황: 처음 시작하거나 결과를 시각적으로 확인하며 작업할 때
- 필요 없는 상황: 반복 작업을 빠르게 실행하거나 세밀한 명령 자동화가 핵심일 때
- 요청문: 나는 초보자야. CLI가 꼭 필요하지 않다면 앱 화면에서 할 수 있는 방법부터 안내해.

### ⬛ 터미널
- 한 줄 설명: 컴퓨터에 짧은 명령을 전달하고 결과를 확인하는 텍스트 작업 창
- 쉽게 비유하면: 버튼 대신 한 줄 명령으로 컴퓨터에 일을 시키는 입력창입니다.
- 왜 필요한가: 설치 확인, 개발 화면 실행, CLI 도구 사용에 필요합니다.
- 사용자가 할 일: AI가 지정한 한 줄을 실행하고 결과 전체를 전달합니다.
- AI가 할 일: 필요한 명령을 만들고 가능하면 직접 실행·검증합니다.
- 추천 상황: Node.js·Git 설치 확인이나 CLI 실행이 필요할 때
- 필요 없는 상황: 단순 아이디어 대화나 화면 피드백만 할 때
- 요청문: 터미널에서 내가 실행해야 하는 명령은 한 번에 하나만 주고, 예상되는 정상 결과도 함께 알려줘.

### ⌨️ CLI
- 한 줄 설명: 앱 대신 터미널에서 대화하고 작업하는 프로그램
- 쉽게 비유하면: 검은 화면에서 코드를 짜는 것이 아니라, AI와 텍스트로 대화하는 또 하나의 창입니다.
- 왜 필요한가: 프로젝트 파일과 컴퓨터 도구를 직접 다루기 좋고 반복 작업을 빠르게 실행할 수 있습니다.
- 사용자가 할 일: 프로젝트 폴더에서 CLI를 시작하고 권한 요청을 판단합니다.
- AI가 할 일: 파일 수정, 명령 실행, 테스트와 결과 보고를 맡습니다.
- 추천 상황: 큰 프로젝트, 반복 실행, 앱이 느리거나 자원 사용이 클 때
- 필요 없는 상황: 터미널이 큰 부담이고 작은 시제품만 확인할 때
- 요청문: 앱과 CLI 중 이번 작업에 더 쉬운 방법을 비교하고, CLI가 필요하면 설치부터 첫 실행까지만 안내해.

### 🔗 API
- 한 줄 설명: 프로그램이 다른 AI나 서비스에 정해진 형식으로 요청하는 통로
- 쉽게 비유하면: 사람이 웹사이트 버튼을 누르는 대신 프로그램끼리 주문서를 주고받는 창구입니다.
- 왜 필요한가: 이미지·음성 생성, 데이터 조회와 대량 반복 작업을 자동화할 수 있습니다.
- 사용자가 할 일: 계정·결제 범위·API 키와 사용 한도를 관리합니다.
- AI가 할 일: 요청 코드, 오류 처리, 비용 제한과 키 보호를 구현합니다.
- 추천 상황: 프로그램 안에서 AI 기능을 반복 호출해야 할 때
- 필요 없는 상황: 사람이 웹 화면에서 몇 번 실행하는 것으로 충분할 때
- 요청문: [기능]에 API가 꼭 필요한지 판단해. 예상 호출 횟수, 비용 위험, 키 보관 방법부터 설명해.

### 🧩 플러그인
- 한 줄 설명: 기존 AI나 프로그램에 여러 기능을 한 묶음으로 추가하는 확장 부품
- 쉽게 비유하면: 기본 공구함에 특정 작업용 공구 세트를 통째로 추가하는 것입니다.
- 왜 필요한가: 스킬·도구·연결 설정을 함께 설치해 새로운 작업 능력을 제공할 수 있습니다.
- 사용자가 할 일: 출처와 권한을 확인하고 필요한 플러그인만 설치합니다.
- AI가 할 일: 플러그인이 제공하는 기능과 적용 범위를 확인해 사용합니다.
- 추천 상황: 검증된 기능 묶음을 여러 작업에서 계속 사용할 때
- 필요 없는 상황: 한 번뿐인 작업이나 출처를 신뢰할 수 없을 때
- 요청문: 이 플러그인이 추가하는 기능, 요구 권한, 외부 전송 데이터와 제거 방법을 먼저 확인해.

### 🔌 MCP
- 한 줄 설명: AI가 외부 데이터와 작업 도구를 공통 방식으로 연결하는 규격
- 쉽게 비유하면: 각기 다른 기계를 같은 모양의 콘센트로 연결하는 표준입니다.
- 왜 필요한가: GitHub·Drive·데이터베이스 같은 외부 시스템을 AI가 직접 읽거나 조작할 수 있습니다.
- 사용자가 할 일: 연결할 서비스와 최소 권한을 승인합니다.
- AI가 할 일: 연결된 도구를 사용해 최신 정보를 읽고 허용된 작업을 수행합니다.
- 추천 상황: 프로젝트 밖의 서비스와 실시간으로 연결해야 할 때
- 필요 없는 상황: 파일 첨부 하나로 충분하거나 외부 권한이 필요 없을 때
- 요청문: [서비스] MCP를 연결하기 전에 읽기·쓰기 권한, 전송 데이터, 사용하지 않을 때 끄는 방법을 설명해.

### 📘 스킬
- 한 줄 설명: 반복 작업의 순서·참고자료·검증법을 저장한 AI 작업 설명서
- 쉽게 비유하면: 새 직원도 같은 품질로 일하도록 만든 표준 작업 매뉴얼입니다.
- 왜 필요한가: 문서 제작·배포·검증처럼 같은 일을 매번 다시 설명하지 않아도 됩니다.
- 사용자가 할 일: 원하는 결과와 반복해서 지킬 기준을 정합니다.
- AI가 할 일: 저장된 절차를 읽고 같은 순서로 작업·검증합니다.
- 추천 상황: 같은 작업이 세 번 이상 반복되고 절차가 안정됐을 때
- 필요 없는 상황: 아직 방법이 계속 바뀌거나 한 번만 할 때
- 요청문: 이 작업을 먼저 한 번 검증하고, 반복 가치가 확인되면 재사용 가능한 스킬로 정리해.

### 🤖 에이전트
- 한 줄 설명: 목표를 받고 필요한 파일과 도구를 선택해 여러 단계를 수행하는 AI 작업자
- 쉽게 비유하면: 질문에 한 번 답하는 사람이 아니라 결과가 나올 때까지 작업하는 담당자입니다.
- 왜 필요한가: 조사→수정→실행→검증 같은 연결된 일을 맡길 수 있습니다.
- 사용자가 할 일: 목표, 범위, 유지 기능과 완료 기준을 전달합니다.
- AI가 할 일: 필요한 순서를 정하고 도구를 사용하며 결과를 검증합니다.
- 추천 상황: 여러 단계가 있지만 목표가 명확한 작업
- 필요 없는 상황: 정답 한 줄이나 단순 설명만 필요할 때
- 요청문: 목표는 [결과]야. 수정 범위와 완료 기준을 먼저 확인하고 필요한 단계만 수행해.

### 👥 하위 에이전트
- 한 줄 설명: 조사·디자인·코딩·검증처럼 독립 작업을 나누어 맡는 보조 AI
- 쉽게 비유하면: 한 담당자가 모든 일을 순서대로 하지 않고 팀원에게 분담하는 방식입니다.
- 왜 필요한가: 서로 독립적인 작업을 동시에 진행하거나 별도의 검토 관점을 얻을 수 있습니다.
- 사용자가 할 일: 분리 가능한 작업인지와 최종 책임자를 정합니다.
- AI가 할 일: 각 결과를 모아 충돌을 해결하고 하나의 결과로 통합합니다.
- 추천 상황: 큰 자료 조사, 여러 독립 모듈, 별도 검토가 필요할 때
- 필요 없는 상황: 작은 작업이나 같은 파일을 동시에 수정해야 할 때
- 요청문: 독립적으로 나눌 수 있는 작업만 하위 에이전트에 맡기고, 최종 결과는 하나로 검증해.

### 🧰 하네스
- 한 줄 설명: AI 모델을 실제로 일하게 만드는 도구·규칙·메모리·실행 환경의 조합
- 쉽게 비유하면: 두뇌가 모델이라면 하네스는 손·공구·작업 규칙이 갖춰진 작업실입니다.
- 왜 필요한가: 같은 모델도 어떤 앱·CLI·도구 연결을 쓰느냐에 따라 결과와 사용감이 달라집니다.
- 사용자가 할 일: 모델 이름만 보지 않고 필요한 도구와 검증 기능을 비교합니다.
- AI가 할 일: 주어진 환경의 파일·도구·규칙을 활용해 일을 수행합니다.
- 추천 상황: Codex와 Claude Code처럼 비슷해 보이는 작업 도구를 비교할 때
- 필요 없는 상황: 단순 대화 모델의 답변만 비교할 때
- 요청문: 모델 성능만 비교하지 말고 이번 작업에 필요한 파일 접근, 도구 실행, 검증 기능까지 비교해.

### 🔐 OAuth와 권한
- 한 줄 설명: 비밀번호를 직접 넘기지 않고 특정 서비스 접근을 허용하는 로그인 방식
- 쉽게 비유하면: 집 열쇠 전체가 아니라 필요한 방만 열 수 있는 임시 출입증을 주는 것입니다.
- 왜 필요한가: AI와 외부 서비스를 연결할 때 계정 비밀번호 노출을 줄이고 권한을 제한할 수 있습니다.
- 사용자가 할 일: 요청 권한과 대상 계정을 확인하고 불필요하면 취소합니다.
- AI가 할 일: 승인된 범위 안에서만 정보를 읽거나 작업합니다.
- 추천 상황: GitHub·Google·Slack 같은 계정을 AI 도구와 연결할 때
- 필요 없는 상황: 공개 정보 조회나 파일 첨부만으로 충분할 때
- 요청문: 로그인 전에 요청되는 권한을 읽기·쓰기별로 설명하고 최소 권한만 요청해.

### ☁️ 로컬과 클라우드
- 한 줄 설명: 내 컴퓨터에서 실행할지 외부 서버에서 실행할지의 차이
- 쉽게 비유하면: 내 주방에서 직접 만들기와 외부 전문 주방에 주문하기의 차이입니다.
- 왜 필요한가: 비용, 속도, 개인정보, 설치 난이도와 컴퓨터 성능이 달라집니다.
- 사용자가 할 일: 보낼 수 있는 데이터와 설치·비용 범위를 결정합니다.
- AI가 할 일: 요구에 맞는 방식을 비교하고 필요한 환경을 구성합니다.
- 추천 상황: 이미지·음성·AI 모델처럼 컴퓨터 자원과 비용 차이가 큰 기능
- 필요 없는 상황: 일반 웹 화면처럼 실행 위치가 결과에 큰 영향을 주지 않을 때
- 요청문: [기능]을 로컬과 클라우드로 만들 때 설치 난이도, 비용, 속도, 개인정보를 비교해.

### 🧠 세션과 컨텍스트
- 한 줄 설명: AI가 현재 작업에서 읽고 기억하는 대화·파일·규칙의 범위
- 쉽게 비유하면: 회의 테이블 위에 펼쳐둔 자료의 양입니다. 너무 많으면 필요한 내용을 찾기 어려워집니다.
- 왜 필요한가: 긴 대화와 많은 도구가 누적되면 느려지고 오래된 지시가 현재 작업과 충돌할 수 있습니다.
- 사용자가 할 일: 핵심 컨텍스트 문서를 유지하고 작업 단위로 새 세션을 시작합니다.
- AI가 할 일: 현재 필요한 파일만 읽고 오래된 가정은 다시 확인합니다.
- 추천 상황: 프로젝트가 길어지거나 AI가 느려지고 방향을 혼동할 때
- 필요 없는 상황: 짧은 일회성 질문
- 요청문: 현재 작업에 필요한 컨텍스트만 남기고, 오래되거나 충돌하는 지시는 먼저 알려줘.
